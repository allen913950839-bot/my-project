# 🚨 必看 - 立即修复步骤

## 问题原因

您在 Vercel 上**没有配置 `DEEPSEEK_API_KEY` 环境变量**！

代码是正确的，但因为缺少 API Key，DeepSeek 调用失败，降级使用了 Gemini。

## ✅ 修复步骤（5分钟完成）

### 步骤1：添加 DeepSeek API Key

1. 打开 https://vercel.com/dashboard
2. 点击 `game-soul-interactive` 项目
3. 点击顶部 **Settings** 标签
4. 点击左侧 **Environment Variables**
5. 点击 **Add New** 按钮
6. 填写：
   ```
   Key: DEEPSEEK_API_KEY
   Value: sk-d214ca84244b4272a682ad79cbab5778
   ```
7. **重要**：勾选所有环境
   - ✅ Production
   - ✅ Preview  
   - ✅ Development
8. 点击 **Save**

### 步骤2：强制重新部署（清除缓存）

1. 点击顶部 **Deployments** 标签
2. 找到最新的部署记录（最上面那条）
3. 点击右侧的 **三个点 (...)** 菜单
4. 选择 **Redeploy**
5. **取消勾选** "Use existing Build Cache"（这个很重要！）
6. 点击 **Redeploy** 确认
7. 等待部署完成（1-3分钟）

### 步骤3：清除浏览器缓存

部署完成后：

1. 打开 https://game-soul-interactive.vercel.app
2. 按 `Cmd + Shift + R`（Mac）强制刷新
3. 或者使用**无痕模式**打开（最保险）

## ❓ 如果还是不行

### 检查环境变量

1. 进入 Vercel Dashboard → Settings → Environment Variables
2. 确认看到：
   - `GEMINI_API_KEY`
   - `DEEPSEEK_API_KEY` ← **必须有这个！**
   - `KV_URL`
   - `KV_REST_API_URL`
   - `KV_REST_API_TOKEN`

1. Deployments → 点击最新的部署
2. 查看 Build Logs
3. 确认没有错误
4. 确认显示：`✓ built in XXXms`
