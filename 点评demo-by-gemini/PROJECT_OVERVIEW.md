# 🎮 GameEcho - 项目概览

## 🎯 项目愿景

GameEcho将传统游戏评论体验转化为互动式AI驱动对话。用户不再只是发表静态评论，而是与游戏专属的AI角色进行符合角色设定的互动。

## 🎨 UI组件详解

### 1. 首页组件
- **顶部导航**：渐变文字Logo + 搜索栏 + 用户图标
- **英雄区**：带动画的渐变标题
- **公司网格**：4个玻璃态卡片，带悬停光效
- **底部**：玻璃暗色底部，带平台描述

### 2. 游戏弹窗组件
- **背景遮罩**：模糊的暗色覆盖层
- **弹窗容器**：玻璃暗色，带公司主题渐变
- **游戏卡片**：每家公司2张卡片，带封面图

### 3. 游戏详情组件

#### 左栏（AI互动）
- **AI角色卡片**
  - 带呼吸光效的浮动头像
  - 角色名 + 个性标签
  - 动态对话气泡（问候/回复/思考）
  
- **评论输入栏**
  - 多行文本区域
  - 富媒体按钮（图片、麦克风、GIF、发送）
  - 游戏主题色的聚焦光效

#### 右栏（社区）
- **筛选标签**：全部/文字/图片/视频
- **评论动态**：加载时的交错动画
- **评论卡片**
  - 用户信息（头像、名称、时间戳）
  - 内容（文字 + 可选图片/视频标签）
  - 互动按钮（点赞、评论）
  - 带主题色强调的AI回复区域

## 🎭 交互功能

### 动画状态

1. **页面加载**
   - 元素带延迟淡入
   - Y轴滑动动画
   - 渐变色动画

2. **悬停状态**
   - 公司卡片：缩放1.05 + 霓虹光晕
   - 按钮：缩放1.1
   - 游戏卡片：缩放1.02 + 边框光晕

3. **点击/触摸状态**
   - 所有交互元素：缩放0.95-0.98
   - 弹窗：从中心的弹簧动画

4. **AI互动**
   - 思考中：脉动点动画
   - 回复：打字机效果，气泡缩放

## 📊 数据结构

### 公司数据
- id、name、logo、themeColor、description

### 游戏数据
- id、companyId、title、cover、themeColor
- aiPersona：{ name、avatar、greeting、personality }
- reviews：Review[]

### 评论数据
- id、userId、userName、userAvatar
- content、images?、hasVideo?
- timestamp、likes
- aiReply（个性化回复）

## 🚀 用户流程

1. **进入**：在首页看到4张公司卡片
2. **浏览**：悬停查看光效，点击公司
3. **选择**：弹窗打开显示该公司的2款游戏
4. **探索**：点击游戏进入详情页
5. **互动**：
   - 阅读AI问候
   - 在输入栏输入评论
   - 观看AI"思考"（1.5秒动画）
   - 看到个性化AI回复
6. **发现**：浏览带AI回复的社区评论
7. **筛选**：使用标签按内容类型筛选
8. **返回**：点击返回按钮回到首页

## 💡 核心差异点

1. **AI个性**：每款游戏都有独特的AI角色（林克、马力欧、派蒙等）
2. **主题色**：每款游戏都有自己的配色方案应用于UI元素
3. **富媒体**：支持文字、图片和视频内容
4. **实时反馈**：模拟AI思考和回复动画
5. **游戏美学**：带霓虹强调的暗色赛博朋克风格
6. **流畅过渡**：Framer Motion驱动所有动画

## 🎯 设计原则

- **沉浸式**：感觉像在游戏UI内
- **互动式**：每个动作都有视觉反馈
- **个性化**：AI对每条评论独特回复
- **现代化**：玻璃态 + 霓虹 = 当代游戏美学
- **响应式**：在所有设备上无缝运行

## 📱 响应式断点

- 移动端（<768px）：单栏，垂直滚动
- 平板（768-1023px）：调整网格，优化间距
- 桌面（≥1024px）：完整双栏布局
