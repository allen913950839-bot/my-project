# 游戏点评宇宙 - 完整产品文档

============================================================
## 【PART 1｜产品定位与价值说明】

### 产品名称
**GameVerse（游戏点评宇宙）**  
AI驱动的沉浸式游戏点评社区

### 核心价值主张
在AI时代，为C端玩家创造一个全新的游戏点评体验：
- **拟人化交互**：每个游戏都有专属虚拟角色陪你聊游戏，进行回复
- **多模态表达**：用户可以通过文字、图片、语音、GIF、短视频全方位展现游戏体验
- **情感共鸣**：AI角色理解你的情绪，给予个性化回应
- **社交破圈**：玩家间的互评机制，形成良性社区生态

### 产品定位
- **行业定位**：游戏内容社区 + AI虚拟人交互
- **用户定位**：18-35岁活跃游戏玩家
- **场景定位**：游戏选择、体验分享、社区互动

### 核心差异化
1. **虚拟角色陪伴**：不再是冷冰冰的评分系统，而是有温度的角色对话
2. **多模态点评**：突破纯文字限制，10秒短视频+GIF让体验可视化
3. **双向互评**：AI角色点评你的内容，玩家之间也能互评
4. **厂商人格化**：每个厂商都有独特人设，强化品牌认知

============================================================
## 【PART 2｜用户画像与场景】

### 主要用户画像

#### 用户A：硬核玩家「游戏深度体验者」
- **年龄**：25-35岁
- **特征**：每周游戏时长20+小时，追求竞技/剧情深度
- **需求**：深度点评、专业交流、炫技展示
- **使用场景**：通关后录制精彩片段+语音解说，获得虚拟角色认可

#### 用户B：休闲玩家「社交分享者」
- **年龄**：18-25岁
- **特征**：碎片化游戏时间，注重社交和趣味
- **需求**：快速分享、获得互动、发现新游戏
- **使用场景**：拍游戏搞笑瞬间GIF，配简短吐槽，在评论区互动

#### 用户C：观望玩家「决策参考者」
- **年龄**：20-30岁
- **特征**：购买前谨慎，依赖真实玩家评价
- **需求**：多维度了解游戏、看实机演示
- **使用场景**：浏览多个玩家的视频点评，参考虚拟角色的推荐指数

### 核心使用场景

**场景1：新游发布日**
玩家小李看到新游戏上线 → 打开GameVerse → 进入该游戏点评页 → 看虚拟角色「游戏制作人AI」的推荐语 → 浏览玩家上传的首发体验视频 → 决定购买

**场景2：通关后炫技**
玩家小王通关困难Boss → 录制10秒战斗片段 → 上传配语音解说 → AI角色回复"恭喜！这套连招堪称教科书" → 其他玩家点赞互评

**场景3：社区互动**
玩家小张浏览某RPG游戏页 → 看到搞笑BUG的GIF → 在评论区与发布者讨论 → 收到虚拟角色的幽默回应 → 关注该玩家

## 【PART 4｜AI点评系统（核心交互 + 多模态逻辑）】

### 多模态点评能力

#### 1. 文字点评
- **输入**：Markdown编辑器，支持表情、话题标签
- **AI处理**：情感分析、关键词提取、相似内容推荐
- **虚拟角色反馈**：文字回复 + 情绪表情

#### 2. 图片点评
- **输入**：最多9张游戏截图，支持标注
- **AI处理**：游戏场景识别、精彩瞬间标记
- **虚拟角色反馈**："这个落日场景太美了！" + 点赞动画

#### 3. 语音点评
- **输入**：最长60秒语音录制
- **AI处理**：语音转文字、情绪识别（兴奋/失望）
- **虚拟角色反馈**：语音回复（TTS）+ 文字转录

#### 4. GIF点评
- **输入**：上传或录制GIF（3-5秒循环）
- **AI处理**：动作识别（如"击杀瞬间"）
- **虚拟角色反馈**："这个反杀时机抓得太准！"

#### 5. 短视频点评（核心亮点）
- **输入**：10秒内实录视频
- **AI处理**：
  - 自动生成封面
  - 识别精彩时刻（击杀/通关/BUG）
  - 添加AI解说字幕
- **虚拟角色反馈**：视频评级（S/A/B/C）+ 具体点评

### 互评机制

#### 玩家 → 玩家
- **评价维度**：有用性、娱乐性、专业性
- **互动方式**：点赞、评论、分享、关注
- **激励体系**：获得点赞→升级勋章→解锁虚拟角色专属对话

#### AI角色 → 玩家
- **自动点评**：上传内容后1分钟内AI角色响应
- **评级系统**：S-F等级，附带具体建议
- **成长激励**："你的点评质量提升了20%！继续加油！"

#### 玩家 → AI角色
- **提问模式**：向虚拟角色问游戏玩法
- **挑战模式**：与AI角色PK谁的点评更受欢迎

## 【PART 5｜核心功能详细说明】

### 功能1：厂商与游戏选择（首页）
**交互流程**：
1. 进入首页，展示热门厂商卡片（含厂商虚拟角色头像）
2. 点击厂商卡片，展开该厂商旗下游戏列表
3. 每个游戏显示：
   - 游戏封面
   - 专属虚拟角色头像
   - 综合评分（5星制）
   - 最新点评数量

**技术实现**：
- 卡片瀑布流布局
- 悬浮显示虚拟角色3D动画
- 点击平滑过渡动画

### 功能2：游戏点评详情页
**页面结构**：
```
┌─────────────────────────────────────┐
│  游戏Banner + 虚拟角色悬浮          │
├─────────────────────────────────────┤
│  【AI角色推荐语】                   │
│  "这款游戏适合喜欢XX的你！"         │
├─────────────────────────────────────┤
│  [发布点评按钮]                     │
├─────────────────────────────────────┤
│  点评列表（多模态卡片）             │
│  ┌─────────────────────────────┐   │
│  │ 玩家头像 | 昵称 | 时间      │   │
│  │ 【内容区：图/文/视频/GIF】 │   │
│  │ AI角色回复：XXXX           │   │
│  │ [👍 126] [💬 45] [⭐ 收藏] │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

### 功能3：多模态内容发布
**操作步骤**：
1. 点击"发布点评"，弹出创作面板
2. 选择内容类型：
   - 📝 纯文字
   - 🖼️ 图片组（拖拽上传）
   - 🎤 语音录制（实时波形）
   - 🎞️ GIF上传
   - 📹 视频录制/上传（10秒倒计时）
3. 添加标签（自动推荐："战斗系统" "画面" "剧情"）
4. 预览 → 发布

**技术特性**：
- 视频自动压缩至720p
- AI生成推荐标题（可编辑）
- 实时草稿保存

### 功能4：AI智能回复系统
**触发条件**：
- 用户发布点评后30秒内
- 用户@虚拟角色名称
- 检测到高质量内容（自动置顶）

**回复策略**：
```javascript
if (内容情绪 == "正面") {
  回复 = 鼓励语 + 具体赞美 + 推荐相关内容
} else if (内容情绪 == "负面") {
  回复 = 理解共鸣 + 建议 + 邀请详细反馈
} else {
  回复 = 中性感谢 + 引导深入分享
}
```

### 功能5：玩家互评系统
**互评入口**：
- 点评卡片下方"评价这条点评"
- 选择标签：👍有用 / 😂搞笑 / 💡专业 / 🔥精彩

**排序算法**：
- 热度 = 点赞数×1 + 评论数×2 + AI评级×10
- 新人内容有24小时流量扶持

**社交功能**：
- 关注玩家，接收TA的新点评通知
- 私信交流（需双方关注）

### 视频处理pipeline
```
用户上传10秒视频
    ↓

    ├─ 时长校验（≤10秒）

    ↓
```

### 数据库Schema示例
```sql
-- 厂商表
CREATE TABLE publishers (
    id VARCHAR(50) PRIMARY KEY,
    name VARCHAR(100),
    character_name VARCHAR(100),
    avatar_url TEXT,
    personality JSONB,
    created_at TIMESTAMP
);

-- 游戏表
CREATE TABLE games (
    id VARCHAR(50) PRIMARY KEY,
    publisher_id VARCHAR(50) REFERENCES publishers(id),
    name VARCHAR(100),
    character JSONB,
    rating DECIMAL(2,1),
    reviews_count INT,
    created_at TIMESTAMP
);

-- 点评表
CREATE TABLE reviews (
    id UUID PRIMARY KEY,
    game_id VARCHAR(50) REFERENCES games(id),
    user_id UUID,
    content_type VARCHAR(20), -- text/image/video/voice/gif
    content JSONB, -- 存储不同类型内容的结构化数据
    ai_reply JSONB, -- AI角色回复
    likes_count INT DEFAULT 0,
    comments_count INT DEFAULT 0,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 互评表
CREATE TABLE interactions (
    id UUID PRIMARY KEY,
    review_id UUID REFERENCES reviews(id),
    user_id UUID,
    type VARCHAR(20), -- like/comment/share
    content TEXT,
    created_at TIMESTAMP
);
```
